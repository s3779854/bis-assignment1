Resources & Tools

Here we have listed the resources and tools we will be using for our project.

Tool/Resource Name Documentation Description Rationale Version & Cost Alternative Tools
Unity Unity User Manual Unity is an engine built for streamlining the process of game development by creating features to build games in a prebuilt engine. It has a lot of tools for both 2D and 3D games which users can create games without innate knowledge of programming a game engine or anything related to it such as creating a renderer, etc. We have decided to use Unity as it has a lot of in built support for 2D games, compared to something like Unreal Engine which has it’s 2D features not getting support for quite some time. Unity is also very well documented making it easy to research how to implement features or how to solve certain problems we might encounter during the development of our game. Using Unity would also give us some experience in developing games using pre-existing engines which would assist us as our group has an interest in developing games. Unity Student/Personal, cost is free. Unreal Engine
Game Maker
Aseprite Aseprite Help Aseprite is an art program specifically created for sprite work and animation. It has features for creating sprite art as well as animating it. As we are going with sprite/pixel art this program is a good choice as it has a lot of features specifically for creating sprite art, it has features for animation such as being able to compare sprites easily for reference during animations. It also has good features for colouring and creating sprites themselves such as palette options or tiling sprites. Standard version downloaded and compiled off Github, cost is free. Photoshop

Group Contract

This contract is including the expected frequency and duration of weekly meeting, expected tools for collaboration, expected frequency of responses, punishment and address-problem method.

Expectation (5 Ground rules):

1. Every member of the group is required to attend a weekly 90-minute meeting on Microsoft Teams through a video chat session.

2. Every member of the group is required to use Trello to record the daily work process and schedule.

3. Every member of the group should use chat facility within the Teams channel, check important messages of the group channel and response at least once a day.

4. Every member of the group should be responsible to finish the effort distribution which has been discussed and agreed. If not, he or she will be punished by SparkPLUS system.

5. Every member of the group should use Trello card to point out the problem so that everyone can help to address it before the project ends.
